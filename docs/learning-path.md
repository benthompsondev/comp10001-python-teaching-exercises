# Learning Path

This repo is meant to be simple on purpose.

These are beginner Python exercises from my earlier teaching support work. If you are new to Python, the best way to use this repo is to run one small file at a time, change one thing, and rerun it until the behavior makes sense.

## How To Use These Exercises

1. Pick one file from the path below.
2. Read it before running it.
3. Run it from a terminal.
4. Change one small thing.
5. Run it again and explain what changed.

That was the teaching style behind a lot of these examples: small program, one idea, quick feedback.

## Suggested Order

| Step | Files | What to focus on |
| --- | --- | --- |
| 1. Input and types | `input_test.py`, `temperature_function.py`, `miles_function.py` | How `input()` returns text, why conversion matters, and how a small function can hide a calculation |
| 2. Conditions | `grade_converter.py`, `nested_grades.py`, `top_3.py` | Turning rules into `if` / `elif` / `else` logic |
| 3. Functions | `basic_my_sum.py`, `basic_my_min.py`, `basic_my_max.py`, `max_function.py`, `no_return_function.py` | Parameters, return values, and the difference between printing and returning |
| 4. Lists and loops | `input_list.py`, `list_2.py`, `lists_3.py`, `list_4.py`, `planet_list.py` | Building lists, looping through them, sorting, and aggregating values |
| 5. String checks | `phone_checker.py`, `postal_code.py`, `postal_code_edit_1.py`, `string_comparison.py`, `vowel_checker.py` | Checking characters, comparing strings, and validating simple formats |
| 6. Larger practice files | `assignment_2.py`, `assignment_3.py`, `textbook_3.py` | Combining input, logic, functions, loops, and small design notes |

## Good Beginner Practice Prompts

- What happens if the user enters text where the program expects a number?
- Which scripts print a value, and which scripts return a value?
- Which examples would be easier to test if the logic was moved into a function?
- Where could you replace repeated `if` checks with a loop?
- Which scripts should handle invalid input more carefully?

## What This Shows

These files are not meant to be modern production Python. They are beginner exercises, kept mostly in their original shape.

The useful part is the teaching pattern:

- break a problem into a small program
- show the concept directly
- run it in a terminal
- talk through what changed
- make the next version a little cleaner

That is still how I like learning and teaching technical work now: start small, make it run, understand it, then improve it.
